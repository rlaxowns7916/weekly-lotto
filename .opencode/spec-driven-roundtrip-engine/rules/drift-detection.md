# 드리프트 감지 규칙

## 드리프트란

경계 문서(CLAUDE.md, IMPLEMENT.md)와 실제 코드 구현 사이의 불일치.

## 감지 기준

### 경계 간 드리프트

1. **책임 중복**: 서로 다른 경계가 동일한 책임을 수행. 통합하지 말고 드리프트 또는 오픈 질문으로 기록한다.

### CLAUDE.md 드리프트

1. **책임 불일치**: 문서의 책임을 코드가 수행하지 않거나, 코드가 문서에 없는 책임을 수행.
2. **인터페이스 불일치**: 문서의 입출력 계약과 실제 인터페이스가 다름.
3. **제약 위반**: 문서에 명시된 제약을 코드가 위반.
4. **의존성 불일치**: 문서의 의존성 목록과 실제 의존 관계가 다름.
5. **시나리오 불일치**: 문서의 Given-When-Then 시나리오가 코드/테스트로 검증되지 않음.
6. **오류 계약 불일치**: 에러 코드/상태/재시도 가능 여부가 문서와 다름.
7. **비기능 불일치**: SLO/보안/타임아웃/동시성 요구가 코드와 다름.

### IMPLEMENT.md 드리프트

1. **구조 불일치**: 문서의 디렉토리 구조와 실제 파일 레이아웃이 다름.
2. **모듈 불일치**: 문서에 기술된 모듈이 삭제/추가/변경됨.
3. **스택 불일치**: 기술 스택이 변경됨 (라이브러리 추가/제거 등).
4. **설정 불일치**: 환경 변수나 설정값이 변경됨.
5. **관측성 불일치**: 로그/메트릭/트레이싱 포인트가 문서와 다름.
6. **테스트 설계 불일치**: 문서의 테스트 경계/필수 케이스와 실제 테스트가 다름.

## 심각도 분류

| 심각도 | 기준 | 조치 |
|--------|------|------|
| 높음 | 책임/인터페이스 불일치 | 즉시 문서 또는 코드 수정 권장 |
| 중간 | 구조/모듈/스택 변경 | 다음 동기화 시 반영 |
| 낮음 | 사소한 표현 차이 | 기록만 |

## 기록 형식

보고서 내 드리프트 기록은 `report-template.md`의 형식을 따른다.
