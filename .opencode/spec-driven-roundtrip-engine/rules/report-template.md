# SRTE 보고서 형식

## compile 보고서 (문서 -> 코드)

```markdown
# SRTE Compile 보고서

## 대상 경계
- {경계 경로}

## 경계별 결과

### {경계 이름}
- **참조 문서**: CLAUDE.md, IMPLEMENT.md
- **계약 품질 게이트**:
  - 시나리오-테스트 이름 1:1 매핑 가능: {통과|실패}
  - 인터페이스/오류 계약 명확성: {통과|실패}
  - 제약/비기능 규칙 수치화: {통과|실패}
  - 구현 추적성/결정성(조건부): {미적용|통과|실패}
  - 가정 안전성(compile 전용): {통과|실패}
  - 실패 코드: {없음|QG-1|QG-2|QG-3|QG-4|QG-5|QG-6|QG-7|복수}
- **게이트 실패 상세(실패 시 필수)**:
  - `SCN-xxx`: {누락된 단정값 또는 모호한 문장}
  - 인터페이스 계약 누락: {필수/옵션/타입/유효성 중 누락 항목}
  - 오류 계약 누락: {error code/status/retry/발생 조건 중 누락 항목}
  - 제약/비기능 누락: {수치/결정형 규칙 누락 항목}
  - 구현 추적성 누락(`QG-5`): {SCN/구현 파일#심볼/테스트명 또는 파일 계약/변경 규칙 칼럼 누락 항목}
  - 확인 불가 관리 누락(`QG-6`): {`확인 불가 사유`/`확인 경로`/`해소 조건` 누락 항목}
  - 가정 안전성 누락(`QG-7`): {`BLOCKING` 가정 존재 또는 `NON-BLOCKING` 가정 3개 이상}
- **문서 보강 체크리스트(실패 시 필수)**:
  - [ ] 실패한 모든 `SCN-xxx`에 필드/연산자/기대값을 추가했다.
  - [ ] 공개 인터페이스의 타입/필수성/유효성 규칙을 보강했다.
  - [ ] 오류 계약의 code/status/retry/조건을 보강했다.
  - [ ] 제약/비기능 요구를 수치 또는 결정형 규칙으로 보강했다.
  - [ ] (선택 섹션 사용 시) `SCN -> 구현 파일#심볼 -> 테스트명` 매핑을 누락 없이 보강했다.
  - [ ] 가정을 `BLOCKING`/`NON-BLOCKING`으로 재분류하고 안전성 규칙을 충족했다.
- **시나리오-테스트 매핑**:
  - `SCN-001` -> `should...`
  - `SCN-002` -> `should...`
- **행동 변경 파일**:
  - `path/to/file` — {변경 내용 요약}
- **구조 변경 파일**:
  - `path/to/file` — {변경 내용 요약}
- **생성 파일**:
  - `path/to/new-file` — {생성 이유}
- **의존성 영향**: {없음|변경된 경계 의존성/호출 관계 영향}
- **가정**:
  - {없음|`[BLOCKING]` 또는 `[NON-BLOCKING]` 형식의 항목 목록}
- **가정 요약**: {총 N건, BLOCKING M건, NON-BLOCKING K건}
- **결정성 강화 상태**: {없음|권장 보강|적용 완료}
- **오픈 질문**: {없음|문서에서 불명확하여 확인 필요한 사항}
  - 항목 형식(필수):
    - 내용: {질문/확인 불가 항목}
    - 확인 불가 사유: {현재 판정 불가 이유}
    - 확인 경로: {확인할 파일/로그/테스트/담당자}
    - 해소 조건: {종결 기준}

## 검증 결과
- 테스트: {실행 여부, 통과/실패}
- 빌드: {실행 여부, 성공/실패}
- 스키마 버전: {최신(CURRENT 기준)|구버전|누락}
- 드리프트: {발견 항목 또는 "없음"}
```
